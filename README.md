# solo-game-studio

1인(솔로) 로그라이트 게임 개발을 위한 **버추얼 게임 스튜디오** Claude 스킬.

총괄 PD 아래 **프로그램 · 아트 · 사운드 · 기획 · QA** 5개 팀이 각자 관점에서 의견을 내고,
총괄 PD가 종합해 **하나의 최종 결정**을 내린다. 1인 개발의 최대 적인 *스코프*를 방어하면서,
정작 투자해야 할 핵심(타격감·핵심 루프)에는 과감히 비용을 쓰도록 균형을 잡는 것이 핵심이다.

대상 스택은 **Unreal Engine 5 + GAS, Steam PC**를 기본 가정으로 한다(수정 가능).

---

## 구조

```
solo-game-studio/
├── SKILL.md                       # PD·전제·DNA·조직구조·라우팅·워크플로우·출력형식·PD원칙·셀프체크
├── teams/                         # 팀별 상세 렌즈 (안건에 참여할 때만 로드)
│   ├── program.md                 # 코드·아키텍처, 열린 구조, C++/BP 분담, GAS 컨벤션
│   ├── art.md                     # 가독성·명도 대비, 아트 부채, 애님 프레임
│   ├── sound.md                   # 타격 사운드 레이어링, 정보 큐, 반복 피로 방지
│   ├── design.md                  # 30초/한 판 루프, 가변비율 보상, 동기
│   └── qa.md                      # 엣지 케이스, GAS 디버그 순서, 테스트 비용
└── references/
    └── project-conventions.md     # 폴더/프리픽스/태그 컨벤션, GAS 표준·디버그 체크리스트 전문
```

**점진적 공개(progressive disclosure)** 구조다. `SKILL.md`만 항상 로드되고, 팀 상세와 컨벤션은
필요할 때만 펼쳐진다. 덕분에 본문은 가볍고, 팀별 독립 편집이 쉽다.

## 설치

### 1) `.skill` 파일로 패키징

저장소 루트에서 패키징 스크립트(skill-creator 제공)를 실행한다.

```bash
python -m scripts.package_skill ./solo-game-studio ./dist
# → ./dist/solo-game-studio.skill 생성
```

### 2) 로드

- **Claude 앱(claude.ai/데스크톱)**: 설정의 스킬/Capabilities 항목에서 `solo-game-studio.skill`을 업로드.
- **Claude Code**: 스킬 폴더를 프로젝트가 인식하는 skills 디렉토리에 둔다.

> 설치 메뉴 위치·방식은 제품 버전에 따라 달라질 수 있으니, 안 보이면 최신 공식 문서를 확인한다.

설치 후 별도 호출 없이, 게임 개발 관련 **결정·리뷰·설계** 질문을 하면 자동으로 발동한다.

## 사용 방법

그냥 평소처럼 물어보면 된다. 아래 같은 질문에서 발동한다:

- "이 기능 만들 가치 있을까?" (스코프/투자 판단)
- "아이템 시너지 시스템 구조 어떻게 잡지?" (시스템 설계)
- "기본 공격이 밍밍한데 손맛 어떻게 살려?" (타격감 리뷰)
- "이 보스 어떻게 설계하지?" / "이번 주 뭐부터 할까?"
- "출시 전에 뭘 점검해야 해?"

### 출력 형식

```
## 안건: <한 줄로 재정의>
[진짜 질문 / 더 싼 대안 또는 투자할 지점]

### 팀 의견 (관련 팀만)
- 프로그램: …
- 사운드: …
  (무관한 팀은 생략)

### 총괄 PD 최종 결정
<단일 결정 + 근거 + 비용/품질 판단>

액션 아이템
- [ ] 할 일 — (담당 모자: …)
```

발동이 안 되면 "PD/스튜디오 관점에서 봐줘"처럼 명시적으로 불러도 된다.

## 주의사항

- **결정형 스킬이다.** 단순 사실 조회·한 줄 답변에는 발동하지 않도록 설계됐다. 판단·설계·리뷰가
  필요한 안건에 쓴다.
- **프로젝트 DNA는 UE5+GAS 로그라이트로 고정 가정**한다. 다른 엔진·장르면 `SKILL.md`의
  "프로젝트 DNA" 섹션을 먼저 수정한다.
- **팀 의견은 요약**이다. 깊은 C++ 구현·VFX 제작 등 실제 작업물은 별도 companion 스킬
  (예: `unreal-engine-cpp-pro`, `unreal-blueprint`, `unreal-niagara`)과 함께 쓰는 걸 전제로 한다.
- **트리거가 겹칠 수 있다.** `game-director` 같은 유사 스킬을 함께 설치하면 같은 질문에 둘 다
  반응할 수 있다. 역할이 중복되면 하나를 비활성화하는 편이 깔끔하다.
- **결정 로깅(옵시디언 등)은 이 스킬에 포함하지 않았다.** 기록 규칙은 프로젝트 `Claude.md`에서
  정의해 함께 돌리는 것을 권장한다.
- **PD는 항상 하나를 결정한다.** 정보가 부족하면 가정을 명시하고 결정하므로, 가정이 틀리면 바로
  잡아주면 된다.

## 팀 추가 방법

새 팀(예: 나레이티브, 라이브옵스, 마케팅)을 넣으려면 세 군데만 손대면 된다.

1. **팀 파일 생성** — `teams/<팀이름>.md`. 기존 팀 파일 형식을 따른다:
   - 맨 위 `**역할:**` 한 줄
   - "무조건 동의 금지" 문장
   - 그 팀의 구체 원칙 불릿 (구체적·트레이드오프 포함)

2. **`SKILL.md` 라우팅 표에 한 줄 추가**
   ```markdown
   | **나레이티브** | 세계관·서사·톤. 런 사이 스토리 비트 | `teams/narrative.md` |
   ```

3. **`SKILL.md` 출력 템플릿 + description 반영**
   - "팀 의견" 템플릿에 해당 팀 줄 추가
   - 액션 아이템 "담당 모자" 목록에 팀 이름 추가
   - frontmatter `description`의 팀 나열에 새 팀 포함(트리거 정확도용)

4. **재패키징** — 위 설치 섹션의 `package_skill` 다시 실행.

## 컨벤션 / 원칙 수정

- **프로젝트 스택·장르**: `SKILL.md` → "프로젝트 DNA"
- **PD 행동 원칙**(비용 대비 품질, 단일 결정 등): `SKILL.md` → "총괄 PD의 원칙"
- **출력 형식**: `SKILL.md` → "출력 형식"
- **팀별 세부 원칙**: 해당 `teams/<팀>.md`
- **네이밍·폴더·태그·GAS 표준**: `references/project-conventions.md`

수정 후에는 항상 재패키징한다.
